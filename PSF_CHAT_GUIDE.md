# PSF Chat Guide

## Why This Exists

PSF is being developed across multiple chats.

This guide keeps the threads separated so they can stay focused while still updating the same master plan.

## Core Principle

One chat, one lane.

Do not let content, tech, marketing, and strategy all collapse into the same thread if they can be handled separately.

## Chat Lanes

### 1. Master Strategy Chat

Use this chat for:

- project vision
- product definition
- positioning
- high-level business choices
- master plan updates

Update:

- `MASTER_PLAN.md`
- `POSITIONING.md`
- `ROADMAP.md`

### 2. Tech Stack / Workflow Chat

Use this chat for:

- Beehiiv vs Wix vs BYO
- content collection workflow
- scraping helpers
- segmentation structure
- build order

Update:

- `TECH_ROUTE.md`
- `STACK_DECISION.md`
- any workflow-specific notes

### 3. Content Chat

Use this chat for:

- newsletter structure
- recurring sections
- utility vs community vs global balance
- click-through versus inline content

Update:

- `MASTER_PLAN.md`
- `POSITIONING.md`
- any content-specific notes

### 4. Marketing Chat

Use this chat for:

- launch messaging
- comp and giveaway ideas
- QR card copy
- gym promotion mechanics

Update:

- `LAUNCH_PLAN.md`
- `MASTER_PLAN.md`
- `ROADMAP.md`

### 5. Sponsorships / Promotions Chat

Use this chat for:

- sponsor packages
- giveaway sponsors

- monetisation experiments

Update:

- `MASTER_PLAN.md`
- `ROADMAP.md`

### 6. Domains / IP Chat

Use this chat for:

- domain ownership
- email addresses

Update:

- the relevant setup doc for that lane
- `PROJECT_BOUNDARIES.md` if naming or ownership rules change

### 7. Project Management / Orchestration Chat

Use this chat for:

- what gets worked on next
- task sequencing
- dependency tracking
- weekly execution rhythm
- keeping the other lanes unblocked
- maintaining the PSF-local PBO queue
- keeping the team-sized-for-two assumptions visible

Update:

- `PBO_PLAN.md`
- `ROADMAP.md`
- `THREAD_HANDOFF.md`
- `WORKFLOW.md`
- `PROJECT_INDEX.md`
- `MASTER_PLAN.md` when the coordination decision changes the broader project

## Reference Doc Rule

If a chat reaches a useful decision, it should update the relevant PSF doc instead of only leaving the decision in the chat.

## Naming Rule

Use PSF names for PSF docs.

Do not mix in Tinyme naming unless the doc is explicitly about the TBO framework lineage.

## Shortcut Rule

If you are unsure which doc to update, start with:

- `MASTER_PLAN.md`

It is the top-level source of truth.
