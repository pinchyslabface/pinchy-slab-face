# Beehiiv For Josiah

This file is a simple guide for Josiah to understand what Beehiiv is doing in Pinchy Slab Face and what to focus on first.

## Why Beehiiv Matters For PSF

Pinchy Slab Face is currently treating the newsletter and audience as the core asset.

That means Beehiiv is not just "an email tool."

For PSF, Beehiiv is likely to be:

- the signup surface
- the newsletter sending tool
- the subscriber database for early-stage audience work
- the place where audience preferences can be captured and used
- one of the main growth tools for the project

## What Josiah Should Learn First

If Josiah is going to become strong in one area quickly, Beehiiv is a good choice.

The most relevant things to learn first are:

1. publication setup and website basics
2. subscribe pages and landing pages
3. subscriber fields and preference capture
4. segments
5. posts and sending workflow
6. growth features like referrals, recommendations, and boosts
7. automations
8. what the API and webhooks are for, without going too deep too early

## Best First Questions For PSF

When learning Beehiiv for this project, these are strong starter questions:

- How should PSF collect signups?
- What information should PSF ask for at signup versus later?
- Should PSF use a general subscribe page, a market-specific landing page, or both?
- What subscriber preferences are worth collecting early?
- How should PSF segment by city, interest, or climbing level?
- Which Beehiiv growth features are worth using now versus later?
- Which Beehiiv features require a paid plan?
- Which parts of PSF should stay manual for now?

## Features Most Relevant To PSF

### 1. Website / landing pages

Beehiiv includes a Website Builder with a default home page and a default subscribe page.

This matters because PSF may be able to launch quickly without building a separate website first.

What Josiah should look for:

- home page options
- subscribe page options
- custom pages
- branding and copy controls
- whether PSF needs one page or multiple signup paths

### 2. Subscribe forms

Beehiiv can power embedded subscribe forms as well as its own subscribe pages.

This matters if PSF wants to collect signups from:

- a Beehiiv-hosted page
- an external site later
- special campaign pages

What Josiah should look for:

- what fields can be added to forms
- what the signup experience feels like
- where form data ends up
- whether forms can support simple audience qualification

### 3. Preference center and subscriber preferences

This is especially relevant for PSF.

If PSF wants to capture extra information such as city, region, climbing level, or content interests, Beehiiv's preference tools and custom fields may be part of the answer.

What Josiah should look for:

- what readers can manage themselves
- what PSF can store as custom fields
- whether preferences are better collected at signup or after signup
- how those preferences can later affect sends or segments

### 4. Subscriber custom fields

Custom fields are one of the most important Beehiiv features for PSF.

They can help store subscriber information like:

- first name
- city
- state
- climbing level
- indoor vs outdoor interest
- event interest

This matters because PSF may eventually want more relevant emails by market and interest.

### 5. Segments

Segments let you group subscribers based on attributes or behavior.

For PSF, this could later support things like:

- Melbourne readers
- Ballina readers
- gym-focused readers
- event-focused readers
- highly engaged subscribers

Josiah should learn:

- what segments are
- how simple or advanced they can be
- which segmentation ideas matter now versus later

### 6. Posts and newsletter workflow

Beehiiv is also where PSF will likely draft and send newsletters.

Josiah should get comfortable with:

- creating posts
- understanding email vs web post behavior
- previewing sends
- testing
- understanding how the archive works

### 7. Referrals

Beehiiv has a referral program that can reward subscribers for bringing in more subscribers.

This may become useful once PSF has an audience worth activating.

It is probably more useful after the basic signup and sending flow is working well.

### 8. Boosts and recommendations

These are growth and promotion tools.

They may be useful later, but PSF should understand them early.

Josiah should learn:

- what they are
- whether they fit PSF's stage
- how much they cost or what plan they require
- whether they are better for later growth rather than first setup

### 9. Automations

Automations can send people through sequences or workflows after signup or other triggers.

These may be useful later for welcome flows or onboarding, but are probably not the first thing PSF needs.

### 10. API and webhooks

These matter when PSF wants Beehiiv to connect more deeply to custom systems or to its own event workflow.

Josiah does not need to become technical here right away.

But it is useful to understand:

- API = a structured way for software to read or update data in Beehiiv
- webhooks = automatic notifications sent from Beehiiv to another system when something happens

For now, the main goal is simply to know these exist and note where they may matter later.

## Suggested PSF Learning Sequence For Josiah

1. Click through the Beehiiv publication and understand the main areas.
2. Look closely at the website builder, subscribe pages, and forms.
3. Review subscriber profiles, custom fields, and preferences.
4. Learn how segments work.
5. Review how posts are created and sent.
6. Review growth tools: referral program, recommendations, boosts.
7. Learn what automations can do.
8. Learn the basics of API and webhooks only after the above makes sense.
