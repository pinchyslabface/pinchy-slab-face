# Roadmap

## North Star

Build a trusted indoor climbing audience first.

That audience can later support:

- newsletter sponsorships
- gym promotion
- a gym and events directory
- city pass products
- climbing accessories and apparel

## Now

### 0. Establish the PBO operating layer

- keep a single ranked queue of work
- track dependencies, blockers, and handoffs
- make the weekly execution rhythm explicit
- assume the current working team is just Josiah and Mike

### 1. Pick the launch structure

Recommended default:

- geography: multi-city structure from day one
- audience: indoor climbers
- value proposition: “never miss what’s on this week”
- city pages: live city plus coming soon cities
- launch city: Melbourne can be the first live active market

### 2. Define the MVP

- signup page
- welcome email
- weekly newsletter
- manual event curation
- simple subscriber segmentation
- AI-assisted ingestion helpers
- Postgres-backed review and correction workflow

### 3. Build the minimum distribution stack

- one landing page
- one email signup form
- one weekly newsletter format
- one simple way for gyms to submit events
- one launch comp mechanic
- QR-based signup flow for live events
- community content submission path

### 4. Curate the first event inventory

- collect events from Instagram, email, websites, WhatsApp, and direct submissions
- normalize into a simple Postgres-backed event store
- keep the review workflow lightweight and repeatable

## Next

### 5. Add a lightweight public archive

This can be a simple page that shows:

- upcoming events
- city or state filters
- gym links

### 6. Start testing monetisation

Possible first monetisation tests:

- sponsor block in the newsletter
- featured event placement
- gym shout-outs
- giveaway sponsor for launch comps

## Later

### 7. Build the directory layer

Add a structured gym database once the list has traction.

### 8. Launch city pass products

Only after you have enough audience and gym relationships.

### 9. Consider physical products

These are real opportunities, but they should follow audience formation, not precede it.

## Decision Rule

If a task does not help grow the list, improve retention, validate monetisation, or make the project easier to execute, it is probably not the next thing to build.
